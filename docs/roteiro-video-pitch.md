# Roteiro do vídeo pitch

Duração máxima: 4 minutos.

## 0:00 a 0:30 - Apresentação

Olá, meu nome é Guilherme Pereira da Silva e este é o TechFlow Task Manager, um sistema web básico de gerenciamento de tarefas criado para uma startup de logística. O objetivo do projeto é demonstrar, na prática, conceitos de Engenharia de Software, metodologias ágeis, versionamento, testes e integração contínua.

## 0:30 a 1:10 - Metodologia ágil e Kanban

Neste projeto foi usada uma abordagem híbrida, com Kanban para organizar o fluxo de trabalho e práticas leves de Scrum para dividir o desenvolvimento em incrementos. No GitHub Projects, as tarefas foram organizadas nas colunas To Do, In Progress e Done. Essa visualização facilita o acompanhamento do progresso e reduz problemas de comunicação.

## 1:10 a 2:00 - Demonstração do sistema

Agora vou demonstrar o sistema em funcionamento. A aplicação permite cadastrar uma tarefa com título, responsável, prazo, descrição e prioridade. Depois, a tarefa aparece no quadro Kanban. É possível mover a tarefa de A fazer para Em progresso e, depois, para Concluído. Também existe busca textual, filtro por status e filtro por prioridade.

## 2:00 a 2:40 - Testes automatizados

O projeto possui testes automatizados usando Node.js. Eles validam operações como criação, listagem, atualização, exclusão e tratamento de entradas inválidas. Esses testes ajudam a identificar falhas antes que o código seja entregue.

## 2:40 a 3:20 - GitHub Actions

O GitHub Actions foi configurado para executar automaticamente a validação de qualidade e os testes sempre que houver push ou pull request. Isso representa uma prática de integração contínua, aumentando a confiabilidade da entrega.

## 3:20 a 3:45 - Mudança de escopo

Durante a simulação, o cliente solicitou uma melhoria para destacar tarefas críticas. Essa mudança foi registrada no Kanban, documentada no README e implementada em commit próprio, com testes atualizados. Isso mostra como projetos ágeis podem se adaptar sem perder controle.

## 3:45 a 4:00 - Fechamento

Como conclusão, o projeto reforça a importância da Engenharia de Software no mercado de tecnologia, pois une organização, qualidade, documentação e capacidade de adaptação em um processo de desenvolvimento mais confiável.
