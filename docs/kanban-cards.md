# Cards do Kanban

Use estes cards no GitHub Projects, distribuídos nas colunas `To Do`, `In Progress` e `Done`.

| Nº | Card | Descrição | Coluna sugerida |
| --- | --- | --- | --- |
| 1 | Definir escopo inicial | Registrar funcionalidades mínimas do sistema de tarefas. | Done |
| 2 | Criar repositório público | Configurar repositório no GitHub com README inicial. | Done |
| 3 | Implementar modelo de tarefas | Criar persistência em JSON e validações principais. | Done |
| 4 | Implementar API CRUD | Criar rotas para cadastrar, listar, atualizar e excluir tarefas. | Done |
| 5 | Criar interface Kanban | Desenvolver tela com colunas A fazer, Em progresso e Concluído. | Done |
| 6 | Adicionar filtros básicos | Permitir busca por texto e filtro por status. | Done |
| 7 | Criar testes automatizados | Cobrir validação de entradas e fluxo da API. | Done |
| 8 | Configurar GitHub Actions | Rodar testes e verificação de qualidade automaticamente. | Done |
| 9 | Revisar documentação | Completar README, UML e instruções de execução. | In Progress |
| 10 | Preparar vídeo pitch | Gravar apresentação do projeto, testes e CI. | To Do |
