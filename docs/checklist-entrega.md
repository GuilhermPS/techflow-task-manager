# Checklist de entrega

## Repositório GitHub

- [ ] Criar repositório público no GitHub.
- [ ] Enviar o projeto local para o repositório remoto.
- [ ] Confirmar que o README aparece corretamente.
- [ ] Verificar se há pelo menos 10 commits.

## GitHub Projects

- [ ] Criar um Project no GitHub.
- [ ] Criar colunas To Do, In Progress e Done.
- [ ] Cadastrar os cards descritos em `docs/kanban-cards.md`.
- [ ] Adicionar o card da mudança de escopo.

## GitHub Actions

- [ ] Acessar a aba Actions.
- [ ] Confirmar que o workflow CI executou com sucesso.
- [ ] Tirar print da execução aprovada.

## Documento teórico

- [ ] Inserir print do Kanban.
- [ ] Inserir print dos commits.
- [ ] Inserir print do GitHub Actions.
- [ ] Exportar ou entregar em DOCX/PDF.

## Vídeo pitch

- [ ] Gravar demonstração do sistema rodando.
- [ ] Mostrar o Kanban no GitHub Projects.
- [ ] Mostrar os testes e o GitHub Actions.
- [ ] Publicar vídeo no YouTube ou Drive com link público.
